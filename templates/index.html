The Categorme API uses OpenAI to classify and summarize any business based on its domain. Even if you supply an email, we'll attempt to parse out the domain portion and fetch the homepage for richer data.

If no homepage is found, certain fields (like phone_numbers or summary) may be empty.

API Documentation

The Categorme API allows you to categorize businesses, extract contact information, and retrieve metadata based on an email or domain. Follow the steps below to authenticate, send requests, and customize your responses.

1. Authorization

To use the API, you must include your unique API Key in the request headers. The API Key is required for every request and ensures secure access. 

  Headers:
    Content-Type: application/json
    X-API-Key:

Replace <YOUR_API_KEY> with your actual API Key, which you can obtain after registering for the service.

2. Making a Request

To categorize a business, send a POST request to the following endpoint: 

  POST https://api.categorme.com/api/categorize

Include the following fields in the JSON body of your request:

  • input_string (required): An email or domain/URL to be categorized (e.g., "devin@trykeep.com" or "trykeep.com").
  • fields (optional): A list of specific fields to include in the response. If omitted, all fields are returned.
  • custom_categories (optional): An array of categories to override the default category list.

3. Request Example

  Headers:
    Content-Type: application/json
    X-API-Key: 
  
  Body:
  {
    "input_string": "devin@trykeep.com",
    "fields": ["type", "category", "summary"],
    "custom_categories": ["Finance", "Retail", "E-commerce"]
  }

4. Response Structure

The API returns a JSON object containing the requested data. Below is an example response: 

  {
    "type": "email",
    "url": "https://trykeep.com",
    "phone_numbers": ["+1-800-555-1234"],
    "emails": ["info@trykeep.com"],
    "website_title": "Keep Business Solutions",
    "website_description": "Helping businesses grow with smart financial tools.",
    "category": "Finance",
    "summary": "Keep offers financial tools for small businesses to manage their expenses and credit efficiently."
  }

5. Response Fields

Below are the fields that may appear in the response, depending on the fields parameter:

  • type: Indicates if the input was an email or domain.
  • url: The canonical URL for the domain.
  • phone_numbers: An array of phone numbers found on the homepage.
  • emails: An array of email addresses found on the homepage.
  • website_title: The title of the website (from the HTML <title> tag).
  • website_description: A description of the website (from the meta description tag).
  • category: The selected category for the business.
  • summary: A two-sentence summary describing the business and its offerings.

6. Filters

Use the fields parameter to filter the response and include only specific fields. For example: 

  "fields": ["category", "summary", "phone_numbers"]

This will return only the category, summary, and phone numbers in the response.

7. Custom Categories

To use your own categories instead of the default list, include them in the custom_categories parameter. If omitted, the API uses a standard set of categories. 

  "custom_categories": ["Healthcare", "Technology", "Education"]

The API will select the best-matching category from the provided list.

8. Error Handling

If an error occurs, the API returns a JSON object with an error field and an appropriate message. 

  Response Example:
  {
    "error": "Unauthorized or missing X-API-Key"
  }

Common errors include invalid API keys, missing required fields, or network issues.
